Every Archetype deployment is configured through environment variables — a small required core plus a large set of optional overrides for search, storage, security, and dataset vocabulary. This page is a reference for the variables that matter, grouped by component. The three services read config independently: the **frontend** (Next.js) from `frontend/.env`, the **backend** (Django) from `api/config/.env`, and the full-stack **infrastructure** compose from `infrastructure/env_file`. Copy the shipped `*.example` files and edit from there.

New to this? Start with the [Quickstart](/deploy/), then come back to tune your instance. To load a dataset, see [Bring your own corpus](/deploy/corpus/).

## Frontend (`frontend/.env`)

The frontend validates its required variables at build and startup (`lib/env.ts`, `next.config.mjs`) and **throws on any missing one**, so the app fails fast rather than booting misconfigured.

| Variable | Required | Purpose |
| --- | --- | --- |
| `NEXT_PUBLIC_API_URL` | yes | Public (browser-facing) backend base URL. **No trailing slash.** Used for data fetches, media, and API rewrites. |
| `NEXT_PUBLIC_IIIF_UPSTREAM` | yes | Base URL of the Sipi IIIF image server, proxied via `/iiif-proxy`. Trailing slash is stripped automatically. |
| `NEXT_PUBLIC_SITE_URL` | yes | Canonical public site URL for metadata, sitemap, and robots. |
| `CORS_ALLOWED_ORIGINS` | yes | Comma-separated origins allowed on the frontend's own `/api/*` routes. |
| `INTERNAL_API_URL` | no | Server-only base for SSR / server-component / route-handler fetches. Not a `NEXT_PUBLIC_` var — never reaches the browser. Falls back to `NEXT_PUBLIC_API_URL` when unset. |

```env
# frontend/.env
NEXT_PUBLIC_API_URL=https://api.archetype.example.org
NEXT_PUBLIC_IIIF_UPSTREAM=https://archetype.example.org/sipi
NEXT_PUBLIC_SITE_URL=https://archetype.example.org
CORS_ALLOWED_ORIGINS=https://archetype.example.org
# Containerized dev only: reach the API from inside the frontend container
# INTERNAL_API_URL=http://host.docker.internal:8000
```

> **Server-vs-client URL split.** Inside a container, `localhost` is the container itself, so browser-facing `NEXT_PUBLIC_API_URL` is unreachable from the Next server. Set `INTERNAL_API_URL` (e.g. `http://host.docker.internal:8000`) to point server-side fetches at a container-reachable host. On host dev and single-origin production, leave it unset.

## Backend (`api/config/.env`)

Django reads config via `django-environ` in `api/config/settings.py`. Copy the shape of `api/config/test.env` for a fresh file.

| Variable | Default | Purpose |
| --- | --- | --- |
| `SECRET_KEY` | insecure placeholder | Signs sessions/tokens. Boot **fails** if unset or left at the `django-insecure…` default while `DEBUG=False`. |
| `DEBUG` | `False` | Debug mode. Off in production (enables HTTPS hardening, secure cookies, HSTS). |
| `DATABASE_URL` | sqlite fallback | Postgres DSN, e.g. `postgresql://user:pass@postgres:5432/db`. |
| `ALLOWED_HOSTS` | `localhost,127.0.0.1` | Comma-separated hostnames Django will serve. |
| `CORS_ALLOWED_ORIGINS` | localhost:3000/8000 | Origins allowed to call the API (scheme + host). |
| `CSRF_TRUSTED_ORIGINS` | localhost:3000/8000 | Trusted origins for CSRF (admin + frontend). |
| `MEILISEARCH_URL` | `http://localhost:7700` | Meilisearch endpoint (compose-internal: `http://meilisearch:7700`). |
| `MEILISEARCH_API_KEY` | empty | Master key; required in production. |
| `MEILISEARCH_INDEX_PREFIX` | empty | Prefix to namespace indexes on a shared Meilisearch. |
| `IIIF_HOST` | `http://localhost:1024/` | Sipi base URL the API embeds in IIIF/thumbnail links. |
| `CELERY_BROKER_URL` / `CELERY_RESULT_BACKEND` | `redis://redis:6379/0` | Celery broker + result store. |
| `CACHE_URL` | `redis://redis:6379/1` | Redis cache backing the search-reindex lock. |
| `DRF_THROTTLE_ANON_RATE` / `DRF_THROTTLE_USER_RATE` | `100/hour` / `1000/hour` | DRF rate limits. |
| `SEARCH_AUTO_REINDEX` | `True` | Auto-reindex on model changes. |

Dataset vocabulary (item types, hair types, repository types, field display names, `SITE_NAME`, `ENABLE_PUBLICATIONS_APP`) is also set here — see [Bring your own corpus](/deploy/corpus/).

```env
# api/config/.env
SECRET_KEY="<long-random-string>"
DEBUG=false
DATABASE_URL="postgresql://postgres:<pass>@postgres:5432/local"
ALLOWED_HOSTS=archetype.example.org,host.docker.internal
CORS_ALLOWED_ORIGINS=https://archetype.example.org
CSRF_TRUSTED_ORIGINS=https://archetype.example.org
MEILISEARCH_URL=http://meilisearch:7700
MEILISEARCH_API_KEY=<meili-master-key>
IIIF_HOST=https://archetype.example.org/sipi
```

> **`DisallowedHost` / `ALLOWED_HOSTS`.** Every hostname that reaches Django — public domain, plus `host.docker.internal` for containerized SSR — must be listed, or requests 400 with `DisallowedHost`. In dev, `MEDIA_ROOT` is `storage/media/` under `api/`.

## Infrastructure (`infrastructure/env_file`)

The full-stack compose (nginx, Postgres, Meilisearch, Redis, Sipi, api, frontend, celery) reads a single `env_file`. Copy `infrastructure/env_file.example`. It carries the backend and frontend values above plus deployment-only knobs:

| Variable | Purpose |
| --- | --- |
| `DOMAIN` | Public domain nginx serves (`localhost` for local). |
| `POSTGRES_DB` / `POSTGRES_USER` / `POSTGRES_PASSWORD` | Postgres container credentials — **keep aligned with `DATABASE_URL`**. |
| `MEILI_ENV` / `MEILI_MASTER_KEY` | `development` (keyless) or `production` with a random key (≥16 bytes). Meilisearch is not exposed on the host. |
| `REDIS_PASSWORD` | Redis auth; when set, include it in the Celery/Redis URLs. |
| `SIPI_JWT_SECRET` (42 chars) / `SIPI_ADMIN_USER` / `SIPI_ADMIN_PASSWORD` | Sipi hardening — override the vendored demo defaults in production. |
| `BACKEND_IMAGE` / `FRONTEND_IMAGE` | Pin container images to an immutable tag or `@sha256:` digest. |

```env
# infrastructure/env_file
DOMAIN=archetype.example.org
POSTGRES_PASSWORD=<pass>
DATABASE_URL="postgresql://postgres:<pass>@postgres:5432/local"
MEILI_ENV=production
MEILI_MASTER_KEY=<meili-master-key>
IIIF_HOST=https://archetype.example.org/sipi
NEXT_PUBLIC_API_URL=https://archetype.example.org
```

> After changing `IIIF_HOST`, run `just reindex` from `infrastructure/` so stored thumbnail links pick up the new host.
