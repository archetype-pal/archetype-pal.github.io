Archetype is an open-source platform for studying manuscripts as evidence: searching a collection, examining images down to the stroke, annotating the hand, linking transcription to the page, and publishing the results in open formats. It is a modern rebuild of the long-running **DigiPal** framework — see [From DigiPal to Archetype](/about/lineage/) — re-architected so that any institution can run it over its own corpus.

The aim is simple: make the close reading of historical handwriting **legible, searchable, and shareable**, without locking that scholarship inside one tool or one website.

## Principles

These are the commitments the project is built around.

- **Open source.** The code is developed in the open and free to use, inspect, and self-host. Nothing essential is hidden behind a hosted service.
- **Standards at the core.** Images use [IIIF](/platform/imaging/); text uses [TEI](/platform/annotation/); annotations can be exported as [W3C Web Annotations](/platform/interoperability/). Work you create is portable and citable by design — no lock-in.
- **Deployment-agnostic.** Archetype is software you run over *your own* manuscripts, on your own servers. One codebase serves many instances; the platform describes what the code can do, never a particular deployment.
- **The scholar is the author.** Human judgement is the record. Editorial work carries an attributed, dated [audit trail](/platform/workflow/), and any future machine assistance is routed through review rather than writing to the record directly (see the [Roadmap](/roadmap/)).
- **Continuity with the scholarship.** The rebuild deliberately preserves the DigiPal method of modelling script — see [Palaeography on Archetype](/about/palaeography/) — while modernising the architecture around it.
