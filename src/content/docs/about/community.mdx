Archetype is developed in the open on GitHub, under the [`archetype-pal`](https://github.com/archetype-pal) organization. It is built to be run and extended by the people who use it — researchers, libraries, museums, and the developers who support them.

## Taking part

There are two kinds of contribution, and both are welcome:

- **Code and infrastructure** — the backend, frontend, and deployment tooling. The [Contributing](/contribute/) guide covers the repository layout, local setup, the architecture guardrails, and the review process. Good first issues are labelled on GitHub.
- **Scholarship and data** — corrections, transcriptions, script descriptions, and feedback from real editorial use. This is often the most valuable input, and it does not require writing code.

If you run an instance over your own corpus, telling the project what worked and what did not is itself a contribution.

## Getting help

- **Questions and bugs** — open an issue in the relevant repository under the [`archetype-pal`](https://github.com/archetype-pal) organization.
- **Deploying an instance** — start with [Running Archetype](/deploy/) and [Troubleshooting](/deploy/troubleshooting/).
- **Using the API** — see [The API](/api/).

We ask everyone taking part to be respectful and collaborative, and to assume good faith.

## How it is governed

Development happens transparently through pull requests and public issue tracking. Changes pass the automated checks and review described in the [Contribution workflow](/contribute/workflow/) before they land. Direction is recorded honestly on the [Roadmap](/roadmap/) and in [What's shipped](/status/), so anyone can see where the project stands.

## Credits & citation

Archetype builds directly on the **DigiPal framework** and roughly fifteen years of digital-palaeography scholarship — see [From DigiPal to Archetype](/about/lineage/). That lineage deserves acknowledgement in any work that uses this platform.

To cite the software, reference the platform by name together with the repository you used (the [`archetype-pal`](https://github.com/archetype-pal) organization on GitHub) and the date or version you worked with; where you rely on the underlying method, cite the DigiPal scholarship as well.

> Archetype is released as open source. Licensing is declared per repository — consult the repository you are using on GitHub for its terms before redistributing.
